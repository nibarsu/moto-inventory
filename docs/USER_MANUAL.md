# Moto Inventory 使用者操作手冊

## 1. 手冊目的

本手冊提供目前系統已完成功能的基本操作說明，適用於日常資料維護、庫存查詢、進貨管理與成本查詢。

目前系統已完成的主要功能：

- 品牌管理
- 商品分類管理
- 倉庫管理
- 供應商管理
- 客戶管理
- 零件商品管理
- 整車商品管理
- 庫存查詢
- 庫存異動
- 庫存調整
- 進貨單管理
- 進貨入庫
- 平均成本查詢
- 銷貨單管理

## 2. 系統登入

1. 開啟系統登入頁。
2. 輸入帳號 Email。
3. 輸入密碼。
4. 按下 `LOG IN` 進入系統。

登入成功後，可在上方導覽列切換各功能模組。

## 4. 基本資料維護順序

建議依下列順序建立基本資料：

1. 品牌管理
2. 商品分類
3. 倉庫管理
4. 供應商管理
5. 客戶管理
6. 零件商品管理
7. 整車商品管理

這樣後續進貨、庫存與成本資料才有完整的主檔可選用。

## 5. 品牌管理

功能用途：

- 建立零件與整車共用品牌資料

主要欄位：

- 品牌代碼
- 品牌名稱
- 英文名稱
- 備註
- 是否啟用

常用操作：

1. 進入 `品牌管理`
2. 點選 `新增品牌`
3. 填寫資料後送出
4. 可於列表中執行查看、編輯、刪除

## 6. 商品分類管理

功能用途：

- 建立商品分類
- 區分零件類與整車類

主要欄位：

- 分類代碼
- 分類名稱
- 類型
  - `part`：零件
  - `vehicle`：整車
- 備註
- 是否啟用

注意事項：

- 零件商品只能選 `part` 類型分類
- 整車商品只能選 `vehicle` 類型分類

## 7. 倉庫管理

- 電話
- 備註
- 是否啟用

主要欄位：

## 12. 庫存查詢

功能用途：

- 查看目前零件與整車在各倉庫的現有庫存

列表內容：

- 類型
- 商品代碼
- 名稱
- 倉庫
- 目前庫存

說明：

- 系統會將零件與整車一起整合顯示
- 目前顯示的是現有庫存餘額

## 13. 庫存異動

功能用途：

- 查看庫存變化紀錄

目前可看到的異動類型：

- `adjust`：人工庫存調整
- `in`：進貨入庫

異動資料通常包含：

- 類型
- 商品
- 倉庫
- 異動前數量
- 異動數量
- 異動後數量
- 備註
- 建立人員
- 時間

## 14. 庫存調整

功能用途：

- 修正現場實際庫存與系統庫存差異

操作步驟：

1. 進入 `庫存調整`
2. 選擇商品類型
   - `part`
   - `vehicle`
3. 選擇商品
4. 選擇倉庫
5. 輸入調整後數量
6. 輸入備註
7. 送出

系統處理方式：

- 會記錄調整前數量
- 會更新調整後數量
- 會自動計算差異數量

範例：

- 原庫存：10
- 調整後：8
- 異動數量：-2

## 15. 進貨單管理

功能用途：

- 建立進貨單主檔與明細

進貨單主檔欄位：

- 進貨單號
- 進貨日期
- 預計到貨日
- 供應商
- 入庫倉庫
- 狀態
- 總額
- 備註

狀態說明：

- `draft`
- `confirmed`
- `completed`
- `cancelled`

### 15.1 新增進貨單

1. 進入 `進貨單管理`
2. 點選 `新增進貨單`
3. 填寫主檔資料
4. 送出後建立進貨單

### 15.2 維護進貨單明細

1. 進入某張進貨單明細頁
2. 點選 `明細管理`
3. 新增或編輯進貨項目

明細可選商品類型：

- 零件
- 整車

明細欄位：

- 商品類型
- 商品
- 數量
- 單價
- 小計
- 備註

系統會自動：

- 儲存商品代碼與名稱快照
- 重算進貨單總額

## 16. 進貨入庫

功能用途：

- 將進貨單尚未到貨的數量實際入庫

操作步驟：

1. 進入 `進貨入庫`
2. 選擇仍有未入庫數量的進貨單
3. 載入明細
4. 輸入本次入庫數量
5. 輸入實際單價
6. 填寫備註
7. 儲存入庫

系統會自動：

- 新增進貨入庫單
- 新增進貨入庫明細
- 增加指定倉庫庫存
- 寫入庫存異動紀錄
- 更新進貨單明細已入庫數量
- 更新商品最近進價
- 更新商品平均成本
- 依入庫完成度更新進貨單狀態

注意事項：

- 已取消的進貨單不可入庫
- 本次入庫數量不可超過剩餘未入庫數量

## 17. 平均成本查詢

功能用途：

- 查看零件與整車的平均成本

查詢條件：

- 類型
  - 全部
  - 零件
  - 整車
- 關鍵字

列表內容：

- 類型
- 代碼
- 名稱
- 品牌
- 分類
- 總庫存
- 最近進價
- 平均成本
- 售價
- 狀態

平均成本說明：

- 目前依進貨入庫資料進行加權平均計算
- 如果原庫存為 0，平均成本會直接採用本次入庫單價

## 18. 銷貨單管理

功能用途：

- 建立銷貨單主檔

銷貨單主檔欄位：

- 銷貨單號
- 銷貨日期
- 預計出貨日
- 客戶
- 出貨倉庫
- 狀態
- 總額
- 備註

狀態說明：

- `draft`
- `confirmed`
- `completed`
- `cancelled`

操作步驟：

1. 進入 `銷貨單管理`
2. 點選 `新增銷貨單`
3. 填寫主檔資料
4. 儲存後可查看、編輯或刪除

注意事項：

- 目前僅完成銷貨單主檔
- 尚未完成銷貨明細
- 尚未完成銷貨出庫
- 尚未影響庫存數量

## 19. 刪除與提示訊息

目前系統提供：

- 刪除前確認視窗
- 儲存成功提示訊息
- 可搜尋下拉選單
- 日期選擇器

如果執行刪除時出現確認視窗，請再次確認資料是否可移除。

## 20. 常見使用建議

### 19.1 建立商品前

請先確認以下主檔已建立：

- 品牌
- 商品分類
- 倉庫
- 供應商

### 19.2 進貨流程建議

建議依序操作：

1. 建立供應商
2. 建立商品
3. 建立進貨單
4. 建立進貨單明細
5. 執行進貨入庫
6. 查詢庫存與平均成本

### 19.3 庫存異常處理

如果現場數量與系統不一致：

1. 先查看 `庫存異動`
2. 確認是否已做入庫或調整
3. 若仍不一致，再透過 `庫存調整` 修正
